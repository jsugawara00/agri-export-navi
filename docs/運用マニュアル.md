# 農産物輸出ナビ 運用マニュアル

作成: 2026-07-18（この日に全経路を実際に動かして検証済み）
対象: 運用者（Jump）。将来手伝ってくれる人が読んでも分かるように書く。

## 0. 覚えることは2つだけ

1. **メールが来たら開く**
2. 本文の案内に従って **「Merge」を押す**か、Claude Codeに**「巡回して」または「巡回PRを一緒に確認して」と言う**

以下はその背景説明。忘れたらここに戻る。

## 1. 自動で動いているもの（何もしなくてよい）

- **クラウド見張り番**（毎週月曜 朝7:00）: 農水省・海外安全HPなど、クラウドから
  届く情報源を自動巡回。変更があれば修正案付きのPRを起票する。
  管理画面: https://claude.ai/code/routines/trig_01JSoK8PbsDY9aHHNaHFpuuG
- **通知係**（GitHub Actions・毎週月曜 朝7:05）: 巡回PRの起票時と、PRが未対応の
  まま残っているとき、@メンション付きコメントを投稿 → 確実にメールが届く
- **Merge→自動デプロイ**: mainにマージされると1〜2分で本番サイトに反映される

## 2. 手動でやること（外務省系の巡回）

外務省サイト（www.mofa.go.jp）はクラウドやCIのIPを拒否するため、
**運用者のPCからしか巡回できない**。

- 手順: Claude Codeを開いて **「巡回して」** と言うだけ（/patrol スキルが走る）
- 頻度: 自由。忘れると**最後の巡回から30日**で「巡回リマインド」のIssueメールが
  届くので、それに従えばよい（=最低月1回のリズム）
- 変化なしなら2分で終わる。変化ありならClaude が差分要約→修正案→PR起票まで行う

## 3. メールが来たときの対応

### 見分け方（差出人はすべて notifications@github.com）

| 本文の名義 | 意味 | やること |
|---|---|---|
| github-actions[bot]＋@メンション | **本命**。巡回PRの案内 or リマインド | 下記の2択 |
| vercel[bot] | プレビューができた等の技術通知 | 読み流してOK |
| （ワークフロー失敗通知） | 通知係自体の不具合 | Claude Codeに相談 |

### 2択の判断（メール本文にも書いてある）

- **承認する場合** → PRを開いて **Merge pull request → Confirm merge**。
  これだけで本番に反映される。マージ前に見た目を確かめたいときは
  vercelコメントの Preview リンクを開く
- **相談する場合** → 何も押さず、Claude Codeに
  **「巡回PRを一緒に確認して」** と言う。差分を見ながら一緒に直して、
  納得してからマージする

### 判断の目安

- 鮮度日付の更新だけ・軽微な文言 → そのままMergeでよい
- **禁止品目・減点数値に関わる変更 → 必ず「相談」**（Claudeも重大変更は
  修正案を出さず「要相談」と書いてくる決まりになっている）

## 5. 原則（このシステムの背骨）

- **自動書き換えは絶対にしない。** AIは検知と下書きまで。反映の引き金は
  常に人間のMerge
- 断定しない。「最終確認は植物防疫所等へ」を全出力に添える
- 網羅を保証しない代わりに、現場の気づきを歓迎し**2〜3日で反映する**
  （利用者は協力者。お願いと約束は必ずセットで）

## 6. 困ったとき

- ルーチンの動きが怪しい → 管理画面（上記URL）で最新実行のログを見る。
  分からなければログをClaude Codeに見せる
- サイトが変 → Vercelのデプロイ一覧を確認（`vercel ls agri-export-navi`）。
  直前のマージが原因ならGitHubでそのPRをRevertできる
- 手元の開発サーバがおかしい → .claude/skills/verify/SKILL.md の手順
  （ポート3000のゾンビプロセス対処）
